1️⃣ STEP 1 — CREATE DASHBOARD AGGREGATION TABLE

Create table:

CREATE TABLE user_finalization_board (
    id INT IDENTITY PRIMARY KEY,
    engagement_id INT,
    user_id INT,
    pending_execution_items INT DEFAULT 0,
    unadjusted_misstatements INT DEFAULT 0,
    kam_items INT DEFAULT 0,
    generated_at DATETIME DEFAULT GETUTCDATE(),
    generated_by INT,
    status VARCHAR(20) DEFAULT 'Generated'
);

This table stores a snapshot of the board at time of generation.

2️⃣ STEP 2 — DEFINE DATA SOURCES (AUTO PULL LOGIC)
A. Pending Items from Execution

Source tables:

execution_workpapers

review_notes

unresolved_issues

Query logic:

SELECT COUNT(*)
FROM execution_workpapers
WHERE engagement_id = @engagementId
AND assigned_to_user_id = @userId
AND status IN ('Open','Pending','Review Required');
B. Unadjusted Differences (ISA 450)

Source:

adjusting_entries

misstatements_register

Logic:

SELECT COUNT(*)
FROM misstatements_register
WHERE engagement_id = @engagementId
AND adjustment_status = 'Unadjusted';
C. Key Audit Matters (ISA 701)

Source:

kam_register

Logic:

SELECT COUNT(*)
FROM kam_register
WHERE engagement_id = @engagementId
AND reporting_required = 1;
3️⃣ STEP 3 — CREATE BOARD GENERATION SERVICE

Create backend endpoint:

POST /api/finalization/generate-board

Server Logic:

1. Get logged-in user ID
2. Pull all counts from source tables
3. Insert record in user_finalization_board
4. Return response to UI
