## Key Architecture Notes
- The DB user is a superuser (bypasses RLS). Tenant isolation relies on withTenantContext() setting app.firm_id + manual WHERE firmId checks.
- clientRoutes.ts is the authoritative client handler (mounted first in routes.ts).
- All admin routes require requireRoles("FIRM_ADMIN", "PARTNER").
- Sentinel errors (NOT_FOUND:, FORBIDDEN:, BAD_REQUEST:) are handled by handleSentinelError() in importRoutes.ts and isqmRoutes.ts.
- Prisma schema is ~13,000 lines. prisma generate needs --max-old-space-size=2048 on constrained environments.
- Dev workflow: NODE_OPTIONS='--max-old-space-size=1024' NODE_ENV=development npx tsx server/index.ts
- Production build: npm run build → node dist/index.cjs
