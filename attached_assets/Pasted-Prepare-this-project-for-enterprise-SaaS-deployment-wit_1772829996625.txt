Prepare this project for enterprise SaaS deployment with the following architecture:

Replit (Development)
→ GitHub (Source Control)
→ GitHub Actions (CI/CD)
→ Docker Image (GitHub Container Registry)
→ Hostinger VPS (Pull Image)
→ Docker Compose
→ Nginx Reverse Proxy
→ SSL (Let's Encrypt)
→ AuditWise SaaS

Perform the following tasks automatically:

1. Dockerization

* Ensure a production-ready Dockerfile for backend (Node.js + Prisma)
* Ensure a Dockerfile for frontend build (Node + Nginx Alpine)
* Use multi-stage builds for smaller images

2. Environment Variables
   Create .env.example containing:
   DATABASE_URL=
   JWT_SECRET=
   ENCRYPTION_MASTER_KEY=
   NODE_ENV=production

3. Docker Compose
   Create docker-compose.yml including services:

db:
image: postgres:16
container_name: auditwise-db
restart: always
environment:
POSTGRES_DB: auditwise
POSTGRES_USER: postgres
POSTGRES_PASSWORD: postgres
volumes:
- pgdata:/var/lib/postgresql/data
ports:
- "5432:5432"

backend:
build: ./backend
container_name: auditwise-backend
depends_on:
- db
env_file:
- .env
restart: always

frontend:
build: ./frontend
container_name: auditwise-frontend
restart: always

nginx:
image: nginx:alpine
container_name: auditwise-nginx
ports:
- "80:80"
- "443:443"
volumes:
- ./docker/nginx.conf:/etc/nginx/nginx.conf
depends_on:
- backend
- frontend

volumes:
pgdata:

4. GitHub Actions
   Create workflow .github/workflows/deploy.yml

When code is pushed to main branch:

* Build Docker images
* Push images to GitHub Container Registry
* Tag images:
  ghcr.io/aqeelalamfca-sys/auditwise-backend:latest
  ghcr.io/aqeelalamfca-sys/auditwise-frontend:latest

5. Production Optimizations

* Enable Prisma migration on startup
* Add healthcheck for backend
* Enable Node memory optimization
* Ensure backend waits for database readiness

6. Security

* Ensure secrets are not committed
* Use .env for runtime secrets
* Add .gitignore for node_modules, logs, env

7. Deployment Instructions
   Provide commands for Hostinger VPS to pull and run containers.

Ensure the repository becomes fully compatible with automated SaaS deployment.
