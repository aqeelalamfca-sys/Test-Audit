Build and integrate a complete “AI Opinion Engine” into the existing AuditWise statutory audit platform. The engine must automatically convert imported audit data into structured, review-ready, standards-linked opinion support outputs for statutory audit engagements. It must work as an AI-assisted audit conclusion module, but must never replace auditor judgment. All conclusions must be presented as advisory, reviewable, override-enabled, evidence-linked, and partner-controlled.

The AI Opinion Engine must be fully integrated with the existing modules and database, especially Upload, Trial Balance, GL, AP, AR, Bank, Confirmations, FS Mapping, Draft FS, Checks, Planning, Execution, Working Papers, and final reporting. It must pull data automatically from all linked modules and generate a centralized opinion-support workflow from data intake to final audit conclusion.

Core objective:
Create an end-to-end AI-driven audit opinion support system that:
1. reads and validates imported client data,
2. analyzes financial statement structure and balances,
3. identifies risks of material misstatement,
4. links findings to assertions and audit areas,
5. checks sufficiency and consistency of audit evidence,
6. evaluates unresolved matters, misstatements, disclosures, estimates, going concern, confirmations, and statutory compliance,
7. produces a structured “Opinion Support Memorandum”,
8. recommends whether the evidence appears to support an unmodified opinion, qualified opinion, adverse opinion, or disclaimer, subject always to human review and approval,
9. stores all findings in database tables with full audit trail,
10. shows all results in a user-friendly dashboard and downloadable working paper format.

General design principles:
- Keep the system practical, logical, and easy to use for auditors.
- Make all AI outputs structured, professional, neutral, standards-linked, and evidence-based.
- Never allow AI to issue a final opinion automatically.
- Require reviewer acceptance, override, edit, and sign-off at every critical stage.
- Every AI conclusion must show: source data used, assumptions, reasoning summary, risk level, affected assertion, amount involved, materiality comparison, audit implication, suggested procedures, reviewer status.
- All pages must be linked, real-time updated, and persistent after refresh/logout.
- Add save, draft, refresh, regenerate AI, export, reviewer comment, partner conclusion, and lock-for-finalization actions.
- Use role-based access control. Only authorized roles can approve opinion-level conclusions.
- All outputs must be tenant-isolated, firm-specific, engagement-specific, and period-specific.

Technology and coding requirements:
- Scan the existing codebase first and reuse current architecture, patterns, UI components, database models, auth structure, and tenant model.
- Do not break existing modules.
- Use existing stack conventions in the project.
- If any schema/table/model/API/UI flow is missing, create it properly and connect it end-to-end.
- Ensure production-safe error handling, validation, audit logs, loading states, fallback states, and retry-safe background operations if already supported.
- Ensure secure multi-tenant isolation on every route, query, model, and service.
- Use centralized service layer for AI analysis and opinion scoring logic.
- Keep code modular, maintainable, and standards-oriented.
