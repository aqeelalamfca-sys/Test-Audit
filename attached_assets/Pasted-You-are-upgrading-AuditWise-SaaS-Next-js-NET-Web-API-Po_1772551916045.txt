GOAL (NON-NEGOTIABLE)

A) DATABASE DESIGN — SPLIT PLATFORM VS TENANT DATA
1) PLATFORM tables (Super Admin allowed):

2) TENANT tables (Super Admin must be blocked):

   ALL tenant tables MUST include firm_id uuid NOT NULL.

B) POSTGRES ROW LEVEL SECURITY (RLS) — HARD ISOLATION
For EVERY tenant table:
1) Ensure firm_id exists and is indexed:
   - ALTER TABLE <tenant_table> ADD COLUMN IF NOT EXISTS firm_id uuid NOT NULL;
   - CREATE INDEX IF NOT EXISTS idx_<table>_firm_id ON <table>(firm_id);
2) Enable RLS:
   - ALTER TABLE <tenant_table> ENABLE ROW LEVEL SECURITY;
3) Create a strict policy that only allows rows for current firm context:
   - CREATE POLICY <table>_firm_isolation
     ON <tenant_table>
     USING (firm_id::text = current_setting('app.firm_id', true));
4) Do NOT create any bypass policy for super admin.
5) Ensure database role used by application is NOT a superuser and does NOT have BYPASSRLS.

C) DB SESSION CONTEXT SETTING (MANDATORY)
