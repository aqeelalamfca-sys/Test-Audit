Perform a complete backend and database integrity audit of the AuditWise application running on Node.js with Prisma ORM and PostgreSQL.

Step 1 – System Scan
Analyze the entire backend structure including:

* prisma/schema.prisma
* migrations folder
* database connection (.env DATABASE_URL)
* existing PostgreSQL tables
* API controllers and services referencing database models.

Identify whether each model defined in the Prisma schema has a corresponding physical table in PostgreSQL. Detect missing tables such as Subscription, User, Firm, Engagement, Invoice, Billing, or any other models referenced in the backend code.

Step 2 – Database Synchronization
If tables are missing, automatically generate them using the Prisma schema.
Execute the following logical workflow:

1. Validate PostgreSQL connection.
2. Generate Prisma client.
3. Run prisma migrate deploy if migrations exist.
4. If migrations are missing, run prisma db push to create tables directly from schema.
5. Ensure all relationships, indexes, and constraints are applied correctly.

Step 3 – Data Integrity Validation
Verify that all tables contain required columns, primary keys, foreign keys, and indexes defined in schema.prisma.
Check for missing relations between models such as:

* Firm → Users
* Engagement → Firm
* Subscription → Firm
* Invoice → Subscription

Automatically fix schema mismatches if detected.

Step 4 – Backend Compatibility Check
Scan backend code for all Prisma queries and confirm that every referenced model exists in the database.
If any query references a missing table or column, update the Prisma schema and regenerate migrations.

Step 5 – Environment Configuration
Ensure environment variables are configured correctly:
DATABASE_URL="postgresql://postgres:PASSWORD@localhost:5432/auditwise"

Verify the application can establish a successful connection to PostgreSQL.

Step 6 – Deployment Readiness
After database synchronization:

* Regenerate Prisma client
* Build backend
* Restart the server process

Final command sequence executed automatically:

npm install
npx prisma generate
npx prisma migrate deploy || npx prisma db push
npm run build

Step 7 – Validation Output
Return a structured report showing:

* Tables created
* Missing tables fixed
* Relations applied
* Migration status
* Database health status

Do not overwrite existing data unless required.
Only create or update missing schema components needed for the application to run correctly.
