You are acting as a senior DevOps and full-stack engineer responsible for preparing the AuditWise application for production deployment.

Objective:
Convert the current AuditWise Alpha environment into a stable Beta production deployment that runs on a Hostinger VPS using Docker containers and deploys automatically through GitHub.

Tasks:

1. Repository Audit
Scan the entire repository and identify:
- frontend application
- backend API
- Prisma database schema
- docker infrastructure files
- environment configuration

Verify all dependencies are properly defined.

2. Backend Production Setup
Ensure backend service:
- runs on port 5000
- exposes endpoint GET /health
- loads environment variables from .env
- connects to PostgreSQL using DATABASE_URL
- includes error middleware and structured logging

3. Database Layer
Use PostgreSQL container as persistent database.

Verify:
DATABASE_URL format
postgresql://postgres:postgres@auditwise-db:5432/auditwise

Ensure:
- Prisma client generation works
- migrations run automatically
- database persists through Docker volumes

4. Frontend Production Build
Verify frontend build works with:

npm run build

Ensure:
- API base URL points to backend container
- production build runs inside Docker

5. Docker Infrastructure
Create or validate docker-compose.yml containing:

services:
auditwise-backend
auditwise-frontend
auditwise-db
auditwise-redis
auditwise-nginx

Add restart policies:

restart: always

Add health checks for:
backend
database
redis

6. Container Dependencies
Ensure backend waits for database readiness.

Example:

depends_on:
auditwise-db:
condition: service_healthy

7. Nginx Reverse Proxy
Configure nginx container so that:

/api routes to backend:5000
/ routes to frontend:3000

Enable gzip compression and security headers.

8. Environment Variables
Generate .env.example containing:

DATABASE_URL
POSTGRES_USER
POSTGRES_PASSWORD
POSTGRES_DB
REDIS_URL
NODE_ENV
PORT

9. GitHub Deployment Pipeline
Create GitHub Actions workflow:

.github/workflows/deploy.yml

Pipeline steps:
1 build containers
2 push images
3 SSH deploy to VPS
4 run

cd /opt/auditwise
git pull origin main
docker compose up -d --build

10. Production Validation
Verify system starts successfully with:

docker compose up -d

Test endpoints:

/health
/api/status

Goal:
AuditWise must deploy automatically from GitHub to VPS, run inside Docker containers, store persistent data, and serve a live website through nginx.
