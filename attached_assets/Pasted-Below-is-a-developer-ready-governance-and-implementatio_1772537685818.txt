Below is a **developer-ready governance and implementation structure** for your **AI-Based Audit Software (Multi-Tenant Architecture)** with clear segregation of:

* **Super Admin (Platform Level)**
* **Firm Admin (Tenant Level)**
* **Users (Role-Based within Firm)**
* **AI API Control Framework**

This is structured for implementation in a **.NET + React + SQL (Multi-Tenant SaaS Model)** environment.

---

# 1️⃣ SYSTEM ARCHITECTURE MODEL

## A. Multi-Tenant Structure (Firm-Based Isolation)

Each firm operates as an independent tenant:

```
Platform (Super Admin Layer)
   ├── Firm A (Admin + Users)
   ├── Firm B (Admin + Users)
   ├── Firm C (Admin + Users)
```

Data Isolation:

* Firm-level database partitioning (Row-level security OR Separate schema per firm)
* FirmID as mandatory foreign key in all operational tables

---

# 2️⃣ ROLE HIERARCHY STRUCTURE

## LEVEL 1 – SUPER ADMIN (Platform Owner)

**Scope:** Entire Application (All Firms)

### Powers & Rights

✔ Create / Edit / Delete Firms
✔ Activate / Suspend / Terminate Firms
✔ Assign Trial Period
✔ Control Subscription & Billing
✔ Push System Updates
✔ Send Global Pop-ups
✔ Send Firm-specific Alerts
✔ Reset Firm Admin Credentials
✔ Configure Default AI API Key
✔ Monitor Usage Analytics
✔ View Audit Logs (All Firms)
✔ Force Logout Users
✔ Lock Specific Modules
✔ Version Control & Upgrade Deployment

---

## LEVEL 2 – FIRM ADMIN (Tenant Admin)

**Scope:** Only Their Firm

### Powers & Rights

✔ Create / Edit / Suspend Users
✔ Assign Roles (Preparer / Reviewer / Approver)
✔ Control Engagement Access
✔ Upload Firm Logo & Branding
✔ Add Firm-Specific AI API Key
✔ Configure Audit Templates
✔ View Firm Analytics
✔ Approve Internal Sign-Off
✔ Set Password Policies
✔ Export Firm Reports

Cannot:
✖ Access Other Firms
✖ Change Billing Plan
✖ Override Super Admin Policies

---

## LEVEL 3 – USERS (Within Firm)

Roles within Firm:

* Preparer
* Reviewer
* Approver
* Read-only

Permissions controlled by Firm Admin.

# STEP 7 – AI API CONTROL STRUCTURE

## A. Default AI API (Controlled by Super Admin)

Super Admin adds:

```
AIConfig Table
- DefaultAPIKey
- ModelName
- TokenLimit
- CostPerToken
- IsActive
```

All firms automatically use this unless overridden.

---

## B. Custom AI API (Optional – Firm Level)

Firm Admin can:

✔ Add Own OpenAI / Azure / Gemini Key
✔ Set Token Limit
✔ Enable / Disable Custom AI

AI Priority Logic:

```
IF Firm.CustomAI = TRUE
   Use FirmAPI
ELSE
   Use DefaultAPI
```

Super Admin can disable custom AI for lower plans.

---
