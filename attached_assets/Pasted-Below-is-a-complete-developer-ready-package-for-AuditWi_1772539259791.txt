# 3) Status Transition Diagram (Firm + Engagement + Workpaper)

## 3.1 Firm / Subscription (Platform Control)

```mermaid
stateDiagram-v2
  [*] --> TRIAL
  TRIAL --> ACTIVE: payment_success OR trial_convert
  TRIAL --> EXPIRED: trial_end_reached
  ACTIVE --> PAST_DUE: payment_failed
  PAST_DUE --> ACTIVE: payment_cleared
  ACTIVE --> SUSPENDED: superadmin_suspend OR policy_violation
  PAST_DUE --> SUSPENDED: grace_period_over
  SUSPENDED --> ACTIVE: superadmin_reactivate
  ACTIVE --> TERMINATED: superadmin_terminate
  TERMINATED --> [*]
```

## 3.2 Engagement Lifecycle (Firm Control)

```mermaid
stateDiagram-v2
  [*] --> DRAFT
  DRAFT --> IN_PROGRESS: start_work
  IN_PROGRESS --> REVIEW: reviewer_requests_review
  REVIEW --> APPROVAL: reviewer_complete
  APPROVAL --> COMPLETED: partner_approves
  REVIEW --> IN_PROGRESS: changes_requested
  APPROVAL --> IN_PROGRESS: partner_changes_requested
  COMPLETED --> ARCHIVED: archive
```

## 3.3 Workpaper Sign-off (Preparer/Reviewer/Approver)

```mermaid
stateDiagram-v2
  [*] --> OPEN
  OPEN --> PREPARED: preparer_complete
  PREPARED --> REVIEWED: reviewer_complete
  REVIEWED --> APPROVED: approver_complete
  PREPARED --> OPEN: reviewer_changes_requested
  REVIEWED --> OPEN: approver_changes_requested
  APPROVED --> LOCKED: engagement_completed OR lock_policy
```

# 4) SaaS Subscription Logic Flow (Enforcement + Feature Flags)

## 4.1 Runtime Enforcement Order (Every Login + API call)

```mermaid
flowchart TD
  A[Request] --> B{Authenticated?}
  B -- No --> X[401]
  B -- Yes --> C{Firm Status}
  C -- SUSPENDED/TERMINATED --> Y[403 Firm Locked]
  C -- TRIAL/ACTIVE --> D{Subscription Status}
  D -- EXPIRED/SUSPENDED/PAST_DUE --> Z[403 Billing Lock + Message]
  D -- ACTIVE/TRIAL --> E{Plan Feature Flags}
  E --> F{Allowed Module?}
  F -- No --> R[403 Feature Not In Plan]
  F -- Yes --> G{RBAC + Firm Scope + Engagement Scope}
  G -- Fail --> S[403 Forbidden]
  G -- Pass --> H[Proceed + Audit Log]
```

## 4.2 Billing Locks (Typical)

* **PAST_DUE**: allow login, show persistent banner + restrict AI + restrict exports
* **SUSPENDED**: block login except FirmAdmin (optional) for payment page only
* **TERMINATED**: read-only for limited window (optional) then archive

# 5) API Architecture Structure (Next.js + .NET)

## 5.1 High-Level

* **Next.js**: UI + server actions (optional) + auth pages
* **.NET API**: business logic + RBAC + tenant enforcement
* **Postgres**: data
* **Worker**: jobs (trial expiry, invoice generation, reminders, AI usage aggregation, backups triggers, audit log compaction)

## 5.2 Suggested API Modules (Routes)

### Auth

* `POST /api/auth/login`
* `POST /api/auth/refresh`
* `POST /api/auth/logout`
* `GET  /api/auth/me`

### Platform (SuperAdmin only)

* `POST /api/platform/firms`
* `PATCH /api/platform/firms/{firmId}` (activate/suspend/terminate)
* `POST /api/platform/firms/{firmId}/trial` (set/extend)
* `GET  /api/platform/subscriptions`
* `POST /api/platform/notifications` (global/firm)
* `PUT  /api/platform/ai/default-key`
* `GET  /api/platform/audit-logs`

### Tenant (FirmAdmin)

* `GET  /api/tenant/settings`
* `PUT  /api/tenant/settings`
* `POST /api/tenant/users`
* `PATCH /api/tenant/users/{userId}`
* `POST /api/tenant/ai/key` (firm override)
* `GET  /api/tenant/audit-logs`

### Engagements

* `POST /api/engagements`
* `GET  /api/engagements`
* `GET  /api/engagements/{id}`
* `PATCH /api/engagements/{id}`
* `POST /api/engagements/{id}/team`
* `POST /api/engagements/{id}/signoff/prepare`
* `POST /api/engagements/{id}/signoff/review`
* `POST /api/engagements/{id}/signoff/approve`

### Workpapers & Evidence

* `POST /api/workpapers`
* `PATCH /api/workpapers/{id}`
* `POST /api/workpapers/{id}/evidence`
* `DELETE /api/evidence/{id}` (restricted)

### AI

* `POST /api/ai/workpaper/{workpaperId}/draft`
* `POST /api/ai/workpaper/{workpaperId}/summarize`
* `POST /api/ai/risk-assistant`
* `GET  /api/ai/usage` (firm admin & super admin scopes)

## 5.3 Middleware (Must-have)

* **TenantResolver**: derives `firmId` from JWT; rejects null firmId unless SuperAdmin
* **SubscriptionGuard**: checks plan/lock state
* **RBACGuard**: role + scope + engagement_team membership
* **AuditLogger**: writes `audit_logs` for every write endpoint and sensitive read endpoints
* **RateLimiter**: strict for `/auth/*` and `/ai/*`

# 7) Production Security Checklist (Go-Live)

## 7.1 Infrastructure

* [ ] SSH key auth only; disable password auth
* [ ] Non-root deploy user + least privilege
* [ ] UFW: allow 22, 80, 443 only
* [ ] Fail2ban enabled
* [ ] OS auto-security updates enabled
* [ ] Docker daemon not exposed publicly

## 7.2 TLS / Network

* [ ] TLS certificates valid + auto renew verified
* [ ] HSTS enabled (after confirming stable HTTPS)
* [ ] Nginx security headers configured
* [ ] Upload limit set (e.g., 50MB) + server-side validation

## 7.3 Application Security

* [ ] JWT signing key strong (64+ chars) and rotated policy documented
* [ ] Refresh tokens stored securely; revoke on password change
* [ ] Brute force protection on login (rate limit + lockout)
* [ ] Role & tenant enforcement in middleware (no “trust client”)
* [ ] All write actions logged in `audit_logs`
* [ ] Soft delete for key entities (users, workpapers, evidence)

## 7.4 Data Protection

* [ ] Firm AI keys encrypted at rest (AES-GCM) using `ENCRYPTION_MASTER_KEY`
* [ ] DB backups encrypted + restore test performed
* [ ] PII minimization (store only what needed)
* [ ] Evidence files stored with hash + immutable pathing per firm

## 7.5 SaaS Controls

* [ ] Trial expiry job working + tested
* [ ] Billing lock states enforced at API gate
* [ ] Plan feature flags enforced server-side
* [ ] AI usage metering per firm (tokens/cost) + throttles

## 7.6 Observability

* [ ] Central logs (at least docker logs + rotation)
* [ ] Uptime alerts enabled (email/telegram)
* [ ] DB disk space alert threshold set
* [ ] Error tracking (optional but recommended): Sentry or similar
