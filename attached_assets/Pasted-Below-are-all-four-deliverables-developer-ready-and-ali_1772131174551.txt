### B) Enforced workflow rules (server-side)

* Only **Prepared** items can be submitted to **In Review**.
* **Approve** requires:

  * all mandatory procedures executed,
  * all required evidence attached and validated,
  * all review notes resolved/closed (or explicitly waived with Partner override + log).
* Any change after approval → auto:

  * increment `version_no`,
  * set previous version `is_current_version=false`,
  * revert status to `IN_PROGRESS`,
  * create signoff_event `VERSION_BUMP`.

## 4) AI Evidence Validation Logic Structure (Inspection-safe)

### A) Inputs (what AI reads)

* Procedure metadata (objective, assertions, risk rating, materiality, sampling parameters)
* Evidence metadata + file text (if parseable)
* GL/TB balances + mapping
* Prior-year comparisons (optional)
* Review notes + exceptions history

### B) Core checks (deterministic first, AI second)

**Layer 1 — Rules (no AI):**

* Required fields present (type/source/date/linkage)
* Evidence attached for mandatory procedures
* Date validity (within period / after year-end where required)
* Cross-reference completeness (FS head → procedure → evidence)
* Sampling coverage threshold met (if applicable)

**Layer 2 — AI scoring (assist, not decide):**

* **Reliability score** (0–100) using factors:

  * external > internal,
  * independent > management-generated,
  * original > copy,
  * direct knowledge > hearsay,
  * controls reliance indicators.
* **Sufficiency score** (0–100) considering:

  * risk level,
  * materiality,
  * coverage breadth (population, sample size),
  * corroboration (multiple sources).
* **Contradiction detection:**

  * highlight mismatches (amounts, dates, counterparties) vs GL/TB.
* **Relevance classification:**

  * evidence supports which assertion(s) (existence, completeness, accuracy, valuation, rights/obligations, presentation).

### C) Outputs (stored + visible)

* evidence_item.reliability_score / sufficiency_score
* flags_json:

  * `missing_required`, `out_of_period`, `amount_mismatch`, `unsupported_assertion`,
  * `contradictory_evidence`, `low_reliability`, `needs_manager_review`
* Suggested next action:

  * “Request third-party confirmation”
  * “Increase sample size”
  * “Perform alternative procedure”
  * “Add management representation / reconcile variance”

### D) Safety & compliance constraints

* AI **never auto-approves**.
* If AI flags HIGH risk contradictions → mandatory reviewer note required before approval.
* All AI outputs labelled:

  * “AI-assisted assessment — subject to professional judgment”
* Every AI run logs:

  * input IDs, model version, timestamp, user, and diff of changes.
