You are building a **Super Admin Enterprise Monitoring Dashboard** for the AuditWise SaaS platform.

Create a **single-page dashboard named “Enterprise-Grade Checklist”** that will appear in the **Super Admin Portal**.

Objective:
Provide a visually powerful operational control panel that displays **live system health from the Hostinger VPS deployment environment**.

Architecture:
Frontend → React (Next.js)
Backend → Node.js Express API
Server Monitoring → SSH execution to VPS
Real-time updates → WebSocket or polling every 20 seconds

Page Location:
Super Admin → System Monitoring → Enterprise-Grade Checklist

Page Design:
Modern enterprise SaaS dashboard using:

• TailwindCSS
• animated progress bars
• status indicator badges
• animated GIF icons
• color coded health indicators
• hover tooltips
• gradient cards
• smooth animations
• responsive grid layout

Color Status System:

Green → Healthy
Orange → Warning
Red → Error
Blue → Information
Grey → Unavailable

Layout Structure:

HEADER SECTION

Display:

System Name: AuditWise Deployment Monitor
Environment: Production
Server Provider: Hostinger VPS
Server IP
Deployment Branch
Current Commit
Server Uptime

MAIN CHECKLIST GRID

Create large animated cards showing live status for:

Repository Status
Deployment Pipeline
Server Infrastructure
Application Runtime
Continuous Deployment
Security & Connectivity
Live Application Health

Each card must include:

status indicator icon
progress level bar
timestamp of last check
animated indicator when updating

SECTION 1 — SOURCE REPOSITORY

Display:

Git repository URL
remote origin
latest commit
commit author
commit timestamp
branch synchronization status

Commands executed:

git log -1
git status

SECTION 2 — DEPLOYMENT PIPELINE

Visual step tracker showing:

Repository Pull
Dependency Installation
Build Status
Application Restart
Cron Deployment Task

Each step shows:

progress percentage
success / failure indicator

SECTION 3 — SERVER HEALTH

Display gauges for:

CPU usage

Load average

Commands:

Display runtime status:

SECTION 6 — SECURITY CHECKS

Display:

SECTION 7 — LIVE APPLICATION HEALTH

Perform health checks:

HTTP status response
API ping
frontend load check

Display green/red indicators.

BACKEND API ENDPOINT

Create endpoint:

/api/system-health

This endpoint must execute secure shell commands on the Hostinger VPS and return structured JSON data.

Example response:

{
cpu: "12%",
memory: "2GB / 32GB",
disk: "15%",
uptime: "12 days",
pm2_status: "online",
git_commit: "abc123",
last_pull: "2026-03-06 09:30"
}

AUTO REFRESH

Dashboard must refresh every **20 seconds**.

Show animated indicator when refreshing.

ALERT SYSTEM

If any check fails:

show flashing red banner
display alert icon
store alert in system logs

PERFORMANCE

Dashboard load time < 2 seconds.

SECURITY

Only **Super Admin role** can access this page.

SSH credentials must be stored in environment variables.

FINAL OUTPUT

A **visually attractive real-time operational monitoring dashboard** that allows the Super Admin to instantly confirm the health of:

deployment pipeline
server infrastructure
application services
continuous deployment system.
