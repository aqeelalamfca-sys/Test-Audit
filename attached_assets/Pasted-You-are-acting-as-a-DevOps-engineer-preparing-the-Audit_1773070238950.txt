You are acting as a DevOps engineer preparing the AuditWise application for production deployment.

Objective:
Deploy the full-stack AuditWise application to a Hostinger VPS through GitHub using Docker containers and make the website live and stable.

Tasks:

1. Repository Audit
- Scan the entire project structure.
- Identify frontend, backend, database, and infrastructure files.
- Ensure all dependencies are defined in package.json and lock files.
- Confirm environment variables are referenced through a .env file.

2. Dockerization
- Ensure the repository contains:
  • docker-compose.yml
  • Dockerfile for backend
  • Dockerfile for frontend
  • .dockerignore
- Define services in docker-compose:
  backend (Node/Express) → port 5000
  frontend (React/Next.js) → port 3000
  postgres database → port 5432
  redis cache → port 6379
  nginx reverse proxy → port 80/443

3. Persistent Data
- Configure Docker volumes for PostgreSQL so database data persists after container restart.
- Ensure backend connects using environment variables:
  DATABASE_URL
  REDIS_URL
- Confirm migrations run automatically on startup.

4. Backend Production Setup
- Verify Express server runs on port 5000.
- Add a health endpoint:
  GET /health → returns JSON { status: "ok" }.
- Ensure logging and error middleware are active.
- Ensure all API modules function:
  authentication
  firm management
  client management
  engagement management
  working papers
  audit modules.

5. Frontend Production Build
- Build optimized production frontend.
- Ensure API base URL points to the backend container.
- Ensure the application builds successfully with:
  npm run build.

6. Nginx Reverse Proxy
- Configure Nginx container to route traffic:
  /api → backend
  / → frontend
- Enable gzip compression.
- Enable HTTPS compatibility.

7. GitHub Deployment Pipeline
- Create a GitHub Actions workflow:
  .github/workflows/deploy.yml

Pipeline steps:
1. Build Docker images
2. Push images
3. Deploy to VPS using SSH
4. Run command on VPS:

cd /opt/auditwise
git pull origin main
docker compose up -d --build

8. Environment Configuration
- Generate .env.example with required variables.
- Ensure production variables load correctly inside containers.

9. Production Validation
- Confirm containers start successfully with:
  docker compose up -d
- Validate endpoints:
  /health
  /api/status
- Ensure frontend loads without errors.

10. Deployment Documentation
- Provide deployment instructions including:
  Replit → GitHub push
  VPS pull and container restart
  domain configuration for auditwise.tech.

Goal:
AuditWise must deploy automatically from GitHub to the VPS, run inside Docker containers, persist all database data, and serve the live website through Nginx.
